This project analyzes the IBM Telco Customer Churn dataset using PostgreSQL, SQL, Python, and Scikit-learn. The dataset contains 7,043 customer records with information about contracts, internet service, payment methods, tenure, monthly charges, customer lifetime value, and churn status. I wrote 14 SQL queries in pgAdmin to explore customer behavior, identify churn patterns, find the most common reasons customers leave, compare churn across different customer groups, and locate high-risk customers. The SQL analysis was also used to create features such as tenure group, estimated revenue, month-to-month contract flags, fiber optic flags, electronic check flags, and technical support flags.

The analysis found that customers with month-to-month contracts, fiber optic internet, electronic check payments, shorter tenure, higher monthly charges, and no technical support were more likely to churn. I then used Python to train and compare Decision Tree, Support Vector Machine, and Gradient Boosting models. Gradient Boosting produced the strongest overall results, reaching 80.7% accuracy and an 85.6% ROC-AUC score. I also used K-Means clustering to divide customers into four segments and identified a high-risk cluster with a 48.42% churn rate, helping show which customers should be prioritized for retention efforts.
